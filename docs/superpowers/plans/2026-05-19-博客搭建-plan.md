# 博客搭建实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 零基础上手，搭建双线博客（A线 NotionNext 低代码 + B线 静态站可控），统一「暖绿素白」视觉，国内可访问。

**Architecture:** B线在项目文件夹内直接构建纯静态 HTML/CSS 网站，部署到 Gitee Pages（国内直连）。A线通过 Notion 写作 + NotionNext 渲染 + Vercel/Cloudflare Pages 部署。域名共用，随时切换主线。

**Tech Stack:** HTML + CSS（B线），Notion + NotionNext + Vercel（A线），Gitee Pages（国内托管）

### Task 5: A线 NotionNext 部署指南

**Files:**
- Create: `notion-next/setup-guide.md`

- [ ] **Step 1: 创建部署指南**

```markdown
# NotionNext 博客搭建指南

## 准备

- Notion 账号（notion.so 免费注册）
- GitHub 账号（github.com 免费注册）
- Vercel 账号（vercel.com，用 GitHub 登录）

## 步骤一：复制 Notion 博客模板

1. 登录 notion.so，新建页面「我的博客」
2. 创建数据库（Table - Inline），命名为「文章」
3. 数据库需要这些属性：
   - title（标题，默认已有）
   - slug（网址后缀，Text 类型）
   - status（状态，Select：Idea / Published）
   - tags（标签，Multi-select）
   - date（日期，Date 类型）
   - summary（摘要，Text 类型）

## 步骤二：公开分享 Notion 页面

1. Notion 右上角 Share → Publish → Publish to web
2. 复制公开链接，类似 https://xxx.notion.site/我的博客-abc123def456
3. 其中 abc123def456 就是页面 ID

## 步骤三：部署 NotionNext

1. 打开 vercel.com，用 GitHub 登录
2. New Project → 搜索 NotionNext → 选择 tangly1024/NotionNext
3. 点击 Deploy，等待完成
4. 访问 xxx.vercel.app → 进入设置页面
5. 填入你的 Notion 页面 ID，保存

## 步骤四：配置主题

1. 访问 xxx.vercel.app/settings
2. 主题选择 NEXT 或 HEO
3. 自定义颜色参考「暖绿素白」配色：
   - 背景色：#faf7f0
   - 主文字：#4a3b2f
   - 强调色：#9b7a5e
   - 点缀色：#7a9a6e

## 后续优化

- 绑自定义域名后可通过 Cloudflare Pages 部署（国内更稳定）
- Notion 文章可一键导出 Markdown 备份
```

修改文件后，进入 Gitee Pages → 点击「更新」按钮触发重新部署。
